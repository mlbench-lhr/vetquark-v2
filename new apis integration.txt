New Files Created
1. route.ts
Purpose: Server-side proxy for eye analysis API

Behavior:

POST endpoint accepting { image_base64: string }
Validates payload (non-empty string)
Uses env var PET_DISEASE_EYE_URL (default: Cloud Run URL)
Uses env var PET_DISEASE_EYE_BEARER for Authorization header
Handles non-JSON upstream responses with 502 error
Returns upstream JSON response with original status
2. route.ts
Purpose: Server-side proxy for skin analysis API

Behavior:

Identical to eye route but uses PET_DISEASE_SKIN_URL and PET_DISEASE_SKIN_BEARER env vars
Default URL: Cloud Run skin endpoint
3. ImageCaptureStep.tsx
Purpose: Image capture and analysis step for eye/skin flows

Props:

analysisType: 'eye' | 'skin'
onBack: () => void
onDone: (payload: { previewDataUrl: string; result: any }) => void
Features:

File input with accept="image/*" and capture="environment"
File size validation (max 10MB)
Image preview using FileReader
"Analyze" button that POSTs to appropriate API endpoint
Loading state during analysis
Error handling with toast notifications
Back button navigation
4. ImageResultStep.tsx
Purpose: Display analysis results for eye/skin flows

Props:

analysisType: 'eye' | 'skin'
previewDataUrl: string
result: any
onBack: () => void
Features:

Displays captured image preview
Handles disease_detected === false: shows message
Handles disease_detected === true: shows:
Leading hypothesis name and confidence (as percentage)
Description
Findings as chips
Differential diagnoses (top 5 sorted by confidence)
"Copy result" button copies formatted text summary to clipboard
"Finish" button navigates to /Veterinarian/home
Technical Architecture
Flow Branching:



Identification Step
├─ analysisType === 'urine' → Timer → Review → Report (existing flow)
└─ analysisType === 'eye'/'skin' → ImageCapture → ImageResult (new flow)
API Integration:

Client-side: Components call /api/pet_disease/eye or /api/pet_disease/skin
Server-side: Next.js API routes proxy to Cloud Run endpoints with bearer auth
Environment variables for URLs and tokens (no hardcoded secrets)
State Management:

analysisType stored in IdentificationDraft
Image analysis results stored in wizard component state (not persisted to draft)
Preview data URL stored in wizard component state
